Quick Start

1Initial VPS Setup (One-time per server)

Run this command once when you first set up your VPS:

curl -fsSL https://your-domain.com/install.sh | bash -s -- setup user@vps-ip
Note: Replace user@vps-ip with your actual VPS credentials (e.g., root@123.45.67.89)

This will:

  • Install Docker and dependencies
  • Build and upload the agent binary
  • Configure systemd service
  • Generate SSH key for GitHub
‚ö†Ô∏è Important: After setup, add the displayed SSH public key to your GitHub repository as a Deploy Key.

2Initialize Your App (Per repository)

Navigate to your project directory and run:

cd my-app
curl -fsSL https://your-domain.com/install.sh | bash -s -- init user@vps-ip
Note: Make sure you're in a Git repository with a remote configured.

This will:

  • Clone your repository on the VPS
  • Register the app in the agent's registry
  • Generate a webhook secret
  • Add GitHub webhook (if gh CLI is installed)

3Deploy

Just push to your configured branch:

git push origin main

The agent will automatically:

  • Validate the webhook signature
  • Pull the latest code
  • Rebuild Docker images
  • Restart containers
‚úÖ That's it! Your app is now deployed and will auto-update on every push.

Prerequisites

  • A Linux VPS (Ubuntu/Debian recommended)
  • SSH access to your VPS
  • A Git repository (GitHub, GitLab, etc.)
  • gh CLI (optional, for automatic webhook setup)

How It Works

DockUp consists of two components:

  1. Agent (Server-side): A Go binary running as a systemd service that listens for webhooks
  2. CLI (Client-side): A bash script that provisions repos via SSH

When you push to GitHub, a webhook triggers the agent, which then:

  1. Validates the webhook signature
  2. Pulls the latest code
  3. Runs docker compose build
  4. Runs docker compose up -d

Manual Deployment

You can trigger a manual deployment via HTTP:

curl -H "Authorization: Bearer YOUR_SECRET" \
http://vps-ip:8080/webhook/manual?app=my-app

Configuration

The agent reads from /etc/dockup/registry.json on the VPS:

{
"my-app": {
"path": "/opt/dockup/apps/my-app",
"branch": "main",
"secret": "webhook-secret-key",
"compose_file": "docker-compose.prod.yml"
}
}

Monitoring

View agent logs:

ssh user@vps-ip "journalctl -u dockup -f"

Check agent status:

ssh user@vps-ip "systemctl status dockup"
