# Documentation Directory Rules

## Directory Structure
```
docs/
├── architecture/     # Architecture documentation
├── guides/          # How-to guides
├── api/             # API documentation
└── examples/        # Code examples
```

## Documentation Types

### Architecture Documentation
- System architecture diagrams
- Component descriptions
- Data flow diagrams
- Design decisions
- Technology choices

### Guides
- Step-by-step tutorials
- Setup guides
- Configuration guides
- Troubleshooting guides
- Best practices

### API Documentation
- Endpoint descriptions
- Request/response formats
- Authentication methods
- Error codes
- Rate limits

### Examples
- Real-world use cases
- Code snippets
- Configuration examples
- Integration examples

## Documentation Standards

### File Naming
- Use lowercase with hyphens: `setup-github-app.md`
- Be descriptive and specific
- Use consistent naming patterns

### Content Structure
1. Title (h1)
2. Brief description
3. Prerequisites (if applicable)
4. Main content
5. Examples
6. Troubleshooting (if applicable)
7. Related links

### Code Examples
- Always include complete, working examples
- Show expected output
- Include error handling
- Add comments for clarity
- Test all examples before committing

### Diagrams
- Use Mermaid for diagrams when possible
- Include alt text for images
- Keep diagrams simple and clear
- Update diagrams when architecture changes

## Maintenance

### Review Process
- Review documentation with code changes
- Update examples when APIs change
- Remove outdated information
- Add new examples for new features

### Versioning
- Document version compatibility
- Note breaking changes
- Maintain changelog for docs
- Archive old versions if needed

- README.md - Main project documentation

- FEATURES.md - Feature list
- GITHUB_APP_SETUP.md - GitHub App setup

### Documentation Updates
- Update README when adding features
- Update ROADMAP when completing phases
- Update FEATURES when adding features
- Update guides when processes change
- Keep all docs in sync
